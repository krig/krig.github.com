M-x emacs-notes

Emacs is just right. There is a learning curve, but once you get it, nothing else is comparable. There are so many things that Emacs does right: The programmability and transparency: If you know C-h a you can see any piece of code in it, and except for the low-level core, everything can be overridden. The minibuffer (no dialogs!). Supporting both frames and windows (or windows and tabs/child windows, for you GUI people). tramp.

.emacs

My .emacs is available at github, although it's called .emacs.d/init.el instead (works just as well). It's mostly random pieces of functionality cobbled together from various bits of the web. Most of it found on EmacsWiki.

Here's a completely arbitrary list of stuff I have in there:

  • A color scheme called color-theme-wombat, originally by Lars H. Nielsen, slightly modified by me

  • Some code to put autosaves in a temp directory

  • Setup for ido, the best thing since sliced bread

  • Modes for vala, C#, Javascript, Lua, etc...

  • Tab expansion, so pressing the Tab key will either use hippie-expand to find a matching expansion or indent according to mode (also hitting Enter indents automatically, no need to use C-j)

  • tabbar plus some little hacks to get it to work the way I want

basics

These two commands are critical to avoid getting stuck and going crazy if accidentally starting Emacs:

  • C-x c: exit
  • C-g: cancel current action

If you also know these, you know enough to start using it.

  • C-h t: start the tutorial
  • C-h a: type some keywords, and get help relating to that
  • M-x: call an interactive function
  • C-x f: open a file
  • C-x s: save the current file
  • C-x C-s: save all files
  • C-/: undo
  • C-g C-x u: redo (yes, I know..)

moving around

(M means alt)

  • C-a, C-e: go to the beginning/end of a line
  • C-b, C-f: move one character backwards/forwards
  • C-p, C-n: move one line up/down
  • M-b, M-f: move backward/forward by word
  • C-v, M-v: move one page up/down

emacs and bash

One cool thing about learning to use Emacs is that bash shares many of the keybindings. Thus things like moving point, kill/region and search (history) all work pretty much as you'd expect from Emacs.
