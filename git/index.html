git diff --cached

Something I learned existed just a few days ago: You know how, when you've added some files to the index and are about to commit, you always want to take a last look at the diff to see if there's any retarded mistakes you might have missed? And how git diff only shows the differences between the working copy and the index? Well, git diff --cached to the rescue. Oh, and git diff HEAD compares the working copy to HEAD. But I already knew that.

git pulln

A simple script to do git pull --rebase in multiple repositories at once.

#!/bin/bash
say() {
  echo -e "\033[1m$*\033[m"
}

for P in "$@"; do
  if [ -d $P ]; then
    WD=`pwd`
    cd $P
    say "Pulling $P"
    git pull --rebase
    cd $WD
  fi
done
