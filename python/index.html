Python is a very interesting programming language. The syntax is one of the cleanest I have seen. It is extremely expressive. When writing python, it feels like writing pseudocode. There is no worrying about datatypes, and because the interpreter is so slow you stop worrying so much about performance. That is, it makes the algorithms stand out clearly, and in that sense it actually helps optimization since you spend time finding a cleaner way of expressing a problem rather than minimizing the number of bytes a class takes in memory.

Of course, that's the main issue with Python as well. The overhead, both in runtime performance and memory usage is beyond ridiculous. There is no functional threading support. The API for writing C extensions is ugly and convoluted. The underlying implementation bleeds through in nasty ways to the language itself.

What I'd want is a compiled language that has the syntax of Python, but also has true garbage collection. I'd even accept implicit static typing in some cases.
